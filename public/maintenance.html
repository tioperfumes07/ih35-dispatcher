Relay / Love’s fuel file (CSV or Excel)

Upload Relay “All Transactions” exports here (same place you record expenses). Requires ERP write secret in Settings. After import, rows appear in the table below — use Record to QuickBooks for each line to create QuickBooks Purchases.

How this file maps to QuickBooks

Fuel ledger lines (diesel / DEF / reefer from volume_* and total_price_*) store unit, gallons, $/gal, total, vendor text, work_date, and a suggested QB expense path (same as Relay template). Each Record to QuickBooks creates its own QuickBooks Purchase with DocNumber (Relay doc / load ref / auto id), Vendor, Class = truck, pay-from Bank, and by default an AccountBasedExpenseLineDetail to the expense account that best matches the path (override with the category field above if needed).

Relay staging lines (parallel list in the preview) use category labels aligned to your spreadsheet template — e.g. Diesel → Fuel Expenses:Fuel-Truck Diesel, DEF → Fuel Expenses:Fuel-DEF-Diesel Exhaust Fluid, Reefer → Fuel Expenses:Fuel-Reefer-Diesel, Relay card fee → Bank Charges:BC-Relay Diesel Code Fee. Those rows are for export / reconciliation; they are not sent to the QuickBooks API from this app unless you use a separate sync process.

Sample Relay CSV uses work_date like 4/15/26 — dates are normalized to ISO on import.

IFTA, fuel tax & DOT compliance (roadmap)

IFTA filings need jurisdiction mileage (taxable and total where applicable), fuel gallons by jurisdiction and fuel type, and auditable source data (often four years). GPS/ELD (for example Samsara vehicle stats and trip history) plus fuel purchases coded in QuickBooks are the usual inputs; this hub already ties loads to revenue and rolls maintenance/AP into settlement views.

Next implementation steps: (1) persist Samsara vehicleId on trucks/trailers; (2) nightly job: odometer or GPS miles by state (route polyline + boundary lookup); (3) import fuel transactions from QBO or fuel card CSV; (4) quarterly IFTA worksheet + CSV/PDF export; (5) driver profile table (CDL/med card/drug test dates) linked to drivers.qbo_vendor_id, separate from QBO PII.

In this app today: Fuel planner (Fuel Board), Samsara fleet/HOS, load-level settlement and driver-pay rollup (Accounting → Settlement), and QBO master sync. Run npm run db:migrate after pull for new fleet/driver columns (see migration 011_fleet_driver_compliance.sql).
