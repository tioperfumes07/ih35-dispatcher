======================================================================
1) PROGRAM OBJECTIVE
======================================================================
Build a QuickBooks-familiar Banking and Reports experience inside IH35 TMS
for a single entity only:
  - Company: IH 35 Transportation LLC
  - Bankruptcy context: Chapter 11, debtor in possession
  - One QBO company file
  - Multiple DIP bank accounts
  - Dedicated Relay bank account in QBO

Parity target:
  - Visual parity and output parity with QBO reports
  - Familiar filters and layouts
  - Not strict 1:1 interaction cloning

Execution policy:
  - Queue-first posting to QBO always (never synchronous posting on UI action)
  - Auto-sync worker every 15 minutes + manual sync button
  - All writes audited
  - Admin/Accountant write controls

======================================================================
2) ARCHITECTURAL COMPONENTS
======================================================================
A. Banking Ledger Engine
  - Canonical transaction store for imports (bank statement, Relay, manual)
  - Transaction status lifecycle: uncategorized -> categorized -> reconciled
  - Register views per account + global uncategorized queue

B. Categorization and Matching Engine
  - Rule-based and suggestion-based categorization
  - Supports action types:
    * Create Expense
    * Apply to Bill
    * Bill Payment
    * Transfer
    * Driver Settlement
    * Split Transaction
    * Factoring Advance
  - Confidence score + explanation for suggested matches
  - Dry-run preview before batch rule apply

C. Settlement and Factoring Engine
  - Driver settlements module with manual gross pay source of truth
  - Future-ready load reference field for dispatch integration
  - Factoring advances support one-to-many load/invoice linkage

D. Reconciliation Engine
  - Monthly per-account reconciliation sessions
  - Lock on zero difference only
  - Admin-only reversal with reason code
  - Lock/unlock events logged in audit_log

E. QBO Posting and Sync Engine
  - Queue-first outbox for all QBO writes
  - Retry/dead-letter controls
  - Header queue count indicator
  - Manual sync + scheduler every 15 minutes

F. Reporting Mirror Engine
  - QBO report mirror API layer with cache and source metadata
  - Same report outputs and filters as approved list
  - CSV/PDF export parity for visible filtered results
  - Snapshot parity tests against QBO output

G. Compliance Bridge (Form 425C)
  - Banking summary feed for lines 19-23 and Part 7 actuals
  - One-click import flow from Banking engine

H. Operational Control Plane
  - Dashboard: uncategorized aging, queue failures, unreconciled accounts,
    settlement liabilities, factoring reconciliation status
  - Idempotency protections for imports and categorize actions
  - Period lock support for month-end integrity
